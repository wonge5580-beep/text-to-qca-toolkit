# Technical Note

This tool helps researchers move from raw qualitative text to a QCA-ready dataset. It is designed for settings such as citizen messages, government replies, public consultation comments, and policy feedback, where a researcher has both raw text and conceptual categories such as dissatisfaction, policy demand, co-production request, trust, or responsiveness. The tool does not replace interpretation. Its purpose is to make a transparent first pass that researchers can inspect, adjust, and document.

The required data are simple CSV files. The text dataset should contain one row per case, a case identifier, a text column, and preferably an outcome column. The prototype file should contain `condition_name`, `prototype`, and `type`. Prototypes are short conceptual descriptions written by the researcher. Rows marked `condition` become QCA conditions. Rows marked `outcome` are retained for conceptual reference, while the main QCA outcome is selected from the uploaded text dataset when available.

The scoring stage converts each text and prototype into a shared feature representation. To handle Chinese and English together, the tool uses Chinese characters, Chinese character bigrams, Latin words, and Latin word bigrams. It also uses a small transparent bilingual concept bridge for the demo concepts, linking terms such as complaint, dissatisfaction, policy, cooperate, and response to Chinese expressions such as 投诉, 不满, 政策, 配合, and 回复. It then calculates cosine similarity between each case text and each prototype. The score table is shown directly in the interface so the researcher can see how raw text becomes condition scores. The interface also lists overlapping features as a simple explanation for why a case is close to a prototype.

The calibration stage converts raw similarity scores into set memberships. The default fuzzy-set rule uses three anchors: full non-membership at 0.15, crossover at 0.35, and full membership at 0.55. Scores below the lower anchor become 0; scores above the upper anchor become 1; scores between anchors are linearly interpolated, with the crossover set to 0.5. The crisp-set option uses a user-selected threshold, with scores at or above the threshold coded 1 and lower scores coded 0. These defaults are intentionally conservative because short Chinese texts and English conceptual prototypes may not share many literal features. The researcher should adjust anchors after inspecting score distributions and substantive examples.

The QCA-ready dataset has one row per case, one column per condition, and a clearly identified outcome column. The truth table groups cases by crisp condition membership. For each configuration, the tool reports the number of cases, case IDs, outcome share, consistency, and coverage. Consistency is calculated as the proportion of cases in the configuration that show the outcome for crisp analysis. Coverage is the proportion of all positive outcome cases covered by the configuration. The solution table reports configurations that meet the selected minimum case count and consistency threshold. Weak or contradictory configurations remain visible in the truth table.

The outputs include a similarity score table, calibrated membership table, QCA-ready dataset, truth table, solution configurations, and figure-ready data for a membership heatmap and consistency-coverage plot. The browser interface also displays these tables and offers CSV exports.

The main assumptions are that the prototypes are substantively meaningful, the selected text field contains enough information for classification, and the chosen calibration anchors are defensible for the research design. The main limitations are prototype sensitivity, text ambiguity, small sample instability, and the simple lexical nature of the scoring method. The bilingual bridge improves the demo but should be expanded, audited, or replaced for a new project. With more time, I would add multilingual embedding models, manual validation labels, intercoder comparison, more formal minimization algorithms, and richer explanation tools that show matched passages rather than only overlapping features.
